    1.introduction

    1. SQL 概述

    SQL 最早的版本由 IBM 开发，最初被叫做 Sequel，发展至今已更名为 SQL，且确立了作为标准关系数据库语言的低位。SQL 语言有以下几个部分：

    • 数据定义语言 (DDL): 提供定义关系模式、删除关系以及修改关系模式的命令；
    • 数据操纵语言 (Data-Manipulation Language, DML): 提供从数据库中查询信息、插入元祖、删除或修改元组的能力；
    • 完整性 (integrity): SQL DDL 包括定义完整性的命令；
    • 视图定义 (view definition): SQL DDL 包括定义视图的命令；
    • 事务控制 (transaction control): SQL 包括定义事务开始或结束的命令；
    • 嵌入式 SQL 和 动态 SQL (embedded and dynamic): 定义 SQL 语句如何嵌入到通用编程语言；
    • 授权: SQL DDL 包括定义对关系和视图的访问权限的命令。

    2. SQL 数据定义

    (1). 基本类型

    • $\mathrm{char}(n)$: 定长字符串，等价于 $\mathrm{character}$，不足部分以空格补全，建议使用 $\mathrm{varchar}$
    • $\mathrm{varchar}(n)$: 可变长度字符串，$n$ 为用户指定的最大长度，等价于 $\mathrm{character\ varying}$
    • $\mathrm{int}$: 整数类型，等价于 $\mathrm{integer}$
    • $\mathrm{smallint}$: 小整数类型
    • $\mathrm{numeric}(p, d)$: 定点数，这个数有 $p$ 位数字 (包括符号位)，其中 $d$ 位在小数点右边
    • $\mathrm{real, precision}$: 浮点数与双精度浮点数
    • $\mathrm{float}(n)$: 精度至少为 $n$ 的浮点数

    (2). 基本模式定义

    create table r (
    A1 D1,
    ...
    An Dn,
    <integrity constraint1>,
    ...
    <integrity constraintn>,
);

    $\mathrm{create\ table}$ 命令的通用格式如右侧所示。
    一些基本的修改操作：

    • $\small\mathrm{insert\ into}\ r\ \mathrm{values}\ (v_1,...)$ : 将元组加入到关系中，值的顺序遵循关系模式中属性的顺序
    • $\small\mathrm{delete\ from}\ r$ : 从关系中删除所有元组
    • $\small\mathrm{drop\ table}\ r$ : 删除一个关系及其所有信息
    • $\small\mathrm{alter\ table}\ r\ \mathrm{add}\ A\ D$ : 为已有关系添加属性
    • $\small\mathrm{alter\ table}\ r\ \mathrm{drop}\ A$ : 为已有关系删除属性

    3. 数据库的修改

    (1). 删除

    删除操作以元组为单位，其格式为 $\small\mathrm{delete\ from}\ r\ \mathrm{where}\ P;$。若省略 $\small\mathrm{where}$，则删除所有元组。

    e.g. 从 $\small\mathit{instructor}$ 中删除 位于$\small\mathit{Waston}$ 大楼工作 的教师元组。 

    DELETE FROM instructor
WHERE dept_name IN (
    SELECT dept_name
    FROM department
    WHERE building = 'Waston'
);

    (2). 插入

    插入一个元组：
     $\small\mathrm{insert\ into}\ \mathit{course}\ \mathrm{values(\mathtt{'}CS\mathtt{'},\mathtt{'}DB\mathtt{'},\mathtt{'}Comp.Sci.\mathtt{'},4)};$
    忽略顺序，指定属性：
     $\small\mathrm{insert\ into}\ \mathit{course}\ (\mathit{course\_id,title,dept\_name,credits})\ \mathrm{values(\mathtt{'}DB\mathtt{'},\mathtt{'}CS\mathtt{'},4,\mathtt{'}Comp.Sci.\mathtt{'})};$
    更通常的情况是，在查询结果的基础上插入元组：

    e.g. 让音乐系每个修满 144 分的学生称为音乐系的教师，工资为 18000 美元 

    INSERT INTO instructor
    SELECT ID,name,dept_name,18000
    FROM student
    WHERE dept_name = 'Music' AND total_credit > 144;

    $\small\mathrm{insert\ into}\ ...\ \mathrm{select}\ ...$ 的执行顺序是先查询再插入。

    (3). 更新

    更新语句的格式为 $\small\mathrm{update}\ r\ \mathrm{set}\ ...\ \mathrm{where}\ ...$。

    e.g. 给工资超过 10000 美元的教师涨 3% 工资，其余涨 5%
    可用两个 $\small\mathrm{update\ ...\ when}$ 语句完成，但两条语句的顺序很关键。更好的办法是使用 $\mathrm{case}$: 

    UPDATE instructor
SET salary = 
    CASE
        WHEN salary <= 10000 THEN salary * 1.05
        ELSE salary * 1.03
    END
-- WHEN salary <= 10000

    标量子查询在更新语句中也非常有用。
