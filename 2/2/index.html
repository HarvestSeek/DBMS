    2.basic queries

    1. 查询的基本结构

    本章使用的例子：
     $\small\mathit{instructor}\ (\mathit{\textcolor{black}{ID}, name, dept\_name,salary})$
     $\small\mathit{teaches}\ (\mathit{\textcolor{black}{ID,course\_id,sec\_id,semester,year}})$
     $\small\mathit{course}\ (\mathit{\textcolor{black}{course\_id},title,dept\_name,credits})$
     $\small\mathit{student}\ (\mathit{\textcolor{black}{ID},name,dept\_name,total\_credits})$
     $\small\mathit{takes}\ (\mathit{\textcolor{black}{ID,course\_id,sec\_id,semester,year},grade})$
     $\small\mathit{section}\ (\mathit{\textcolor{black}{course\_id,sec\_id,semester,year},building,room\_number,timeslot\_id})$

    (1). 基本查询

    一个 SQL 查询需要用到三种子句，其基本格式如右侧所示。 

    select A1,A2,...,An,
from r1,r2,...,rm,
where P;
    • $\mathrm{select}\ \small name$ : 投影操作，可以带有算术表达式
      • $\mathrm{select\ \small distinct}\ \small name$ : 去重
      • $\mathrm{select\ all}\ \small name$ : 不去重，也是默认行为
      • $\mathrm{select\ *}$ : 所有的列
    • $\mathrm{from}\ r$ : 输入的关系
    • $\mathrm{where}\ \small predicate$: 选择操作，根据谓词选择元组，允许使用连词 $\mathrm{and、or}$ 和 $\mathrm{not}$
      • $\small\mathrm{between\ ...\ and}$ : 用于简化比较，例如 $\small\mathrm{where\ \mathit{salary}\ between\ 10000\ and\ 90000}$
      • $\small(v_1,...,v_n)$ : 分量的字典序比较，例如 $\small\mathrm{where\ (\mathit{i.ID,\ name})=(\mathit{t.ID,}}$ '$\small abc$'$)$
    尽管各子句必须按 $\small\mathrm{select}$-$\small\mathrm{from}$-$\small\mathrm{where}$ 的次序给出，但其逻辑上的顺序为 $\small\mathrm{from}$-$\small\mathrm{where}$-$\small\mathrm{select}$。

    (2). 自然连接

    对于 $\mathrm{from}$ 子句列出的关系，通过 "," 连接产生笛卡尔积，而通过 $\mathrm{natural\ join}$ 进行自然连接。

    e.g.1 列出教师的名字以及他们所讲授课程的名称 

    SELECT name, title
FROM instructor NATURAL JOIN teaches, course
WHERE teaches.course_id = course.course_id;
    注意不要对 $\mathit{course}$ 进行自然连接，属性 $\mathit{dept\_name}$ 是不被期望相等的。

    可以显式指定自然连接中期望相等的属性，例 1 中的 $\mathrm{from}$ 子句可重写如下 

    FROM (instructor NATURAL JOIN teaches) JOIN course USING (course_id)

    2. 附加运算

    (1). 更名运算

    一般情况下，查询结果的属性名会自动从基关系导出。更名运算的意义在于：

    • $\mathrm{from}$ 中的关系可能存在同名属性
    • 在 $\mathrm{select}$ 中使用算术表达式会使结果属性无名
    • 有时候需要修改结果的属性名
    为此，SQL 提供重命名的方法：$\small\mathit{old\_name}$ $\mathrm{as}$ $\small\mathit{new\_name}$，$\mathrm{as}$ 子句可以出现在 $\mathrm{select}$ 和 $\mathrm{from}$ 中。

    e.g.2 重命名属性 

    SELECT name as instructor_name FROM instructor;
    e.g.3 重命名关系，简化名称 
    SELECT T.name, S.course_id
FROM instructor as T, teaches as S
where T.ID = S.ID;

    重命名另一个应用是比较同一个关系中的元组。

    e.g.4 比生物系某一个教师工资高的所有教师的姓名 

    SELECT DISTINCT T.name
FROM instructor as T, instructor as S
WHERE T.salary > S.salary AND S.dept_name = 'Biology'

    (2). 字符串运算

    SQL 使用单引号标示字符串，通过双单引号转义。

    • 相等: 在 SQL 标准中，相等是大小写敏感的，实际的数据库通常需要自己定义相等行为
    • 字符串函数: SQL 允许字符串上的函数，具体的函数需要查询相应数据库的手册

    SQL 支持字符串的模式匹配。

    • $\mathrm{like}$ 通过两个字符描述模式，允许使用 $\mathrm{not}$
      • $\%$ : 匹配任意子串，例如 '$\small\%\mathrm{Comp}\%$'
      • $\_$ : 匹配任意一个字符，例如 '$\small\mathtt{\_\_\_}\%$' 匹配至少含三个字符的字符串
      • $\mathrm{escape}$ : 用于自定义转义字符，例如 $\small\mathrm{like}$ '$\small\mathrm{\mathtt{\backslash\backslash} ab\mathtt{\backslash} \% cd \%}$' $\small\mathrm{escape}$ '$\small\mathtt{\backslash}$' 匹配 "$\small\mathtt{\backslash}\mathrm{ab\%cd}$"
    • $\mathrm{similar\ to}$: 在 SQL:1999 中引入，语法类似于 UNIX 的正则表达式

    (3). 元组排序

    查询结果可以通过 $\mathrm{order\ by}$ 子句排列显示次序，默认使用升序，也可以通过 $\mathrm{asc}$ 和 $\mathrm{desc}$ 显式指明。

    e.g.5 按工资降序输出教师关系，若工资相同，按姓名升序排列 

    SELECT *
FROM instructor
ORDER BY salary DESC, name ASC

    3. 集合运算

     并、交、差分别通过关键字 $\mathrm{union、intersect}$ 和 $\mathrm{except}$ 实现。集合运算是自动去重的，如果需要保留重复，需要在相应关键字后添加 $\mathrm{all}$。

    4. 空值

    空值给算术运算、比较运算和集合运算带来问题。

    ① 算术运算

    如果算术表达式任一输入为空，那么运算结果为空。

    ② 比较运算

     SQL 将涉及空值的任何比较运算 (例如 $\small 1<\mathit{null}$ ) 的结果视为 $\mathit{unknown}$，它是建立在 $\mathit{true}$ 和 $\mathit{false}$ 外的第三个逻辑值。而对于逻辑运算：

      $\small\mathit{true}$ $\small\mathit{false}$ $\small\mathit{unknown}$
    $\small\mathrm{and}$ $\small\mathit{unknown}$ $\small\mathit{false}$ $\small\mathit{unknown}$
    $\small\mathrm{or}$ $\small\mathit{true}$ $\small\mathit{unknown}$ $\small\mathit{unknown}$
    $\small\mathrm{not}$ $-$ $-$ $\small\mathit{unknown}$

     我们可以通过关键字 $\mathrm{null}$ 测试空值，例如 $\small\mathrm{where}\ salary\ \mathrm{is\ not\ null}$，某些 SQL 允许我们使用 $\small\mathrm{is\ unknown}$。

    ③ 集合运算

     当使用 $\mathrm{select\ distinct}$ 或集合运算判断重复元组时，如果两个元组的某个分量都为 $\mathit{null}$，那么认为这两个分量相等，而不是 $\mathit{unknown}$。

    5. 聚集函数

    (1). 基本聚集

    聚集函数是以一个集合作为输入，返回单个值的函数。SQL 提供了 5 个固有聚集函数：

    • $\small\mathrm{count([all]\ *)}$ : 统计元组个数，不允许使用 $\small\mathrm{distinct}$
    • $\small\mathrm{count([distinct\ |\ all]\ \mathit{attr})}$ : 统计一列中值的个数
    • $\small\mathrm{sum([distinct\ |\ all]\ \mathit{attr})}$ : 计算一列值的总和
    • $\small\mathrm{avg([distinct\ |\ all]\ \mathit{attr})}$ : 计算一列值的平均值
    • $\small\mathrm{max([distinct\ |\ all]\ \mathit{attr})}$、$\small\mathrm{min([distinct\ |\ all]\ \mathit{attr})}$ : 计算一列值的最大、最小值
    其中 $\small\mathrm{sum、avg}$ 只能作用于数字集。

    (2). 分组聚集

    有时候我们也希望聚集函数作用于一组元组集合上，然后分别聚集并构造表，此时需要使用 $\mathrm{group\ by}$。

    e.g.6 查询每个系在 2010 年春季学期讲授一门课程的教师人数 

    SELECT dept_name, COUNT(DISTINCT ID) as instr_count
FROM instructor NATURAL JOIN teaches
WHERE semester = 'Spring' AND year = 2010
group by dept_name;

    如果一个属性没有出现在 $\mathrm{group\ by}$ 中且出现在了 $\mathrm{select}$ 中，那么它一定出现在聚集函数内部，否则这就是一条错误的查询。

    (3). $\mathrm{having}$ 子句

     有时候，我们需要直接对分组限定条件，此时需要使用 $\mathrm{having}$ 子句。由于 $\mathrm{having}$ 子句中的谓词在分组生效后才起作用，因此可以使用聚集函数。
     包含聚集、$\small\mathrm{group\ by}$ 和 $\small\mathrm{having}$ 的查询可通过下面的操作序列定义：

    • 先根据 $\small\mathrm{from}$ 计算关系，并将 $\small\mathrm{where}$ 的谓词应用于此关系
    • 根据 $\small\mathrm{group\ by}$ 指定的属性，把上一步的关系分组
    • 根据 $\small\mathrm{having}$ 子句的条件过滤上一步的分组
    • 最后 $\small\mathrm{select}$ 根据剩下的分组产生结果

    e.g.7 对于在 2009 年每个讲授的课程段，如果该课程段至少有两名学生选课，查询选修该课程段的所有学生的总学分的平均值 

    SELECT course_id, sec_id, semester, year, AVG(total_credits)
FROM student NATURAL JOIN takes
WHERE year = 2009
GROUP BY course_id, sec_id, semester, year
HAVING COUNT(ID) >= 2;

    (4). 空值的聚集

    除了 $\mathrm{count(*)}$ 以外的所有聚集函数都忽略输入集合中的空值。
    $\mathrm{count}$ 空集的结果为 0，其余的聚集函数对于空集的结果为空值。
