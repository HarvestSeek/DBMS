    3.nested subqueries

    1. $\mathrm{where}$ 中的子查询 ($\mathrm{having}$)

     $\mathrm{where}$ 子查询的执行：将外查询表的每一行，代入内查询作为检验，如果内查询返回的结果满足条件，这一行可作为外查询的结果行。
     在 $\mathrm{where}$ 子查询中，在相关名称上可以应用作用域规则，即子查询可用外查询的名称，如果重复定义则内查询的名称有效。使用了外层名称的子查询被称作相关子查询 (correlated subquery).

    (1). 成员资格测试

    连接词 $\mathrm{in}$ 测试元组是否是集合中的成员

    e.g.8 查询 2009 秋和 2010 春同时开课的所有课程
    可以通过集合的交书写，在这里使用嵌套子查询 

    SELECT DISTINCT course_id
FROM section
WHERE semester = 'Fall' and year = 2009 and
        course_id IN (SELECT DISTINCT course_id
                      FROM section
                      WHERE semester = 'Spring' and year = 2010);

    $\mathrm{in}$ 也支持枚举集合和多属性测试。例如 $\small\mathrm{where\ \mathit{name}\ not\ in\ (}$'$\small a$','$\small b$'$\small )$、$\small\mathrm{where\ } (r_1,r_2)\ \mathrm{in\ (select ...}$

    (2). 集合的比较

    • $\mathrm{some}$ : 存在，$\small\mathrm{= some}$ 和 $\small\mathrm{in}$ 等价
    • $\mathrm{all}$ : 任意，$\small\mathrm{<> all}$ 和 $\small\mathrm{not\ in}$ 等价

    e.g.8 找出平均工资最高的系 

    SELECT dept_name
FROM instructor
GROUP BY dept_name
HAVING AVG(salary) >= ALL (SELECT AVG(salary)
                           FROM instructor
                           GROUP BY dept_name);

    (3). 空关系测试

    通过 $\mathrm{exists}$ 测试一个子查询的结果是否存在元组。例 8 也可以写为 

    SELECT course_id
FROM section AS S
WHERE semester = 'Fall' AND year = 2009 and 
    EXISTS (SELECT *
            FROM section AS T
            WHERE semester = 'Spring' AND year = 2010 AND 
                S.course_id = T.course_id);

    可以通过 $\mathrm{not\ exists}$ 实现关系代数中的除法：

    e.g.9 查询选修了生物系开设的所有课程的学生。
    ① 没上过的生物系的课 = 所有生物系的课 - 上过的课 

    SELECT S.ID, S.name
FROM student AS S
WHERE NOT EXISTS ((SELECT course_id
                   FROM course
                   WHERE dept_name = 'Biology')
                   EXCEPT
                  (SELECT T.course_id
                   FROM takes AS T
                   WHERE S.ID = T.ID));
    ② 
    SELECT S.ID, S.name
FROM student 
WHERE NOT EXISTS (SELECT course_id
                  FROM course
                  WHERE dept_name = 'Biology' AND NOT EXISTS
                        (SELECT T.course_id
                         FROM takes
                         WHERE student.ID = takes.ID));

    (4). 重复元组测试

    如果查询结果没有重复元素，$\mathrm{unique}$ 子句返回 $\mathit{true}$。

    e.g.10 查询在 2009 年至少开设两次的课程。

    SELECT T.course_id
FROM course AS T
WHERE NOT UNIQUE (
    SELECT R.course_id
    FROM section AS R
    WHERE T.course_id = R.course_id AND R.year = 2009);

    2. $\mathrm{from}$ 中的子查询

     任何 $\mathrm{select}$-$\mathrm{from}$-$\mathrm{where}$ 表达式返回的结果都是关系，在此观点下，子查询可以插入到任何关系出现的位置，包括 $\mathrm{from}$ 子句。

    e.g.11 查询在所有系中工资总额最大的系。

    SELECT MAX(total_salary)
FROM (
    SELECT dept_name, SUM(salary)
    FROM instructor
    GROUP BY dept_name)
    AS dept_total (dept_name, total_salary);

    • 在此例中，通过 $\mathrm{as}$ 对子查询结果及其属性名均进行了重命名，某些 SQL 实现强制要求对子查询结果重命名；
    • $\mathrm{from}$ 子查询不允许使用来自其他关系的相关变量，而 SQL:2003 允许 $\mathrm{from}$ 中的子查询使用 $\mathrm{lateral}$ 作为子查询的前缀，以访问相关属性。

    3. $\mathrm{with}$ 子句

    $\mathrm{with}$ 子句提供定义临时关系的方法 (可定义多个)，此定义只对包含 $\mathrm{with}$ 子句的查询有效。

    e.g.12 查询预算值最大的系。

    WITH max_budget(value) AS (
    SELECT MAX(budget) FROM department)
SELECT budget
FROM department, max_budget
WHERE department.budget = max_budget.value;

    通过 $\mathrm{with}$ 实现的查询比 $\mathrm{where}$ 或 $\mathrm{from}$ 嵌套的子查询更直观易懂。

    4. 标量子查询

     SQL 允许子查询出现在任何返回单个值的表达式能够出现的地方，只要该子查询一定会返回包括单个属性的单个元组。这样的子查询被称作标量子查询。

    e.g.13 列出所有的系及它们的教师数。

    SELECT dept_name (
    SELECT COUNT(*)
    FROM instructor
    WHERE department.dept_name = instructor.dept_name)
    AS num_instructors
FROM department
